Create Build



A build is a unique iteration of your application with the same general project configuration and signature but different application resources and a different version number to enable app stores distinguish between your builds.



To create a build for your project, follow these steps.



1- Click on the project from the general dashboard. You will be routed to the project dashboard.



2- Click on the Add build button. If your subscription allows for in-app promotions, be sure to set the list option to Builds to reveal the Add build button.




3- The overlay requires you to perform the following actions.


  • Change the icon if the build should have a different app icon. Do this by clicking on the icon card.

  • Insert a Build name as an identifier. This is for your reference and will not be the name of your app on your user's devices.

  • Select a zip file containing all the files of your static web application. After selecting the zip file, the interface will extract the files and require you to select an entry-point for your web application. Locate this file by walking through folders with highlighted dropdown icons.





4- You will receive an email when the build is completed. You may also monitor the build progress on the dashboard. When the build is completed, you can download the built resources by clicking on the relevant icons.
